# 系统自省机制

## 发现来源
用户洞察：审视系统本身可以优化的地方，也是进化机制的一部分。

## 机制
在 AGENTS.md 中加入"系统自省"环节：

```
每隔一段时间（或每次重大工作后），问自己三个问题：

1. 这个工作台有没有让我觉得不方便的地方？
   → 有 → 记录到 lab_book/mistakes/ 或直接修复

2. 有没有什么重复劳动可以自动化？
   → 有 → 写成脚本放进 tools/

3. 有没有什么知识散落各处应该整合？
   → 有 → 整理并更新结构
```

## 与现有机制的关系
```
事前回顾  → 用旧知识指导新行动
事后优化  → 用新经验更新旧知识
系统自省  → 用使用者的感受改进工具本身
```

三个机制构成完整的进化三角。
