# 知识类项目的通用结构

## 发现来源
构建 political_philosophy 项目时发现

## 模式
知识类项目（不是代码项目）都适用这个三层结构：

```
inbox/        → 一切入口（现象、问题、想法）
knowledge/    → 沉淀（规律、案例、框架）
connections/  → 关系网（条目之间的连接）
```

## 为什么有效
- inbox 降低记录门槛——先扔进来再说
- knowledge 强制加工——不是堆信息，而是提炼
- connections 让知识产生化学反应——孤立的知识没有价值

## 适用范围
- 政治哲学研究
- 历史学习
- 读书笔记
- 技术调研
- 任何"需要从大量信息中提炼规律"的场景

## 验证
political_philosophy 项目是第一个验证。后续观察是否适用于更多知识类项目。
